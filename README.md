# Курсовая работа: Рекомендательная система друзей в социальной сети
## Тема 20: Анализ графа социальной сети с алгоритмом BFS

### Описание проекта
Данный проект представляет собой приложение на языке C# с графическим интерфейсом Windows Forms, разработанное в рамках курсовой работы по дисциплине «Алгоритмизация и программирование». Приложение реализует систему рекомендации новых друзей в социальной сети на основе анализа графа связей. Бизнес-логика вынесена в отдельную библиотеку классов, интерфейс и код разделены, проект покрыт модульными тестами на фреймворке xUnit.

### Функциональность

**Основная механика** — Поиск друзей друзей через обход графа  
Для заданного пользователя система находит всех друзей его друзей (пользователей на расстоянии 2), которые ещё не являются его друзьями. Используется алгоритм поиска в ширину (BFS) для обхода графа социальных связей.

**Ранжирование кандидатов**  
Рекомендации сортируются по количеству общих друзей с целевым пользователем. Дополнительно рассчитывается коэффициент связности — отношение количества общих друзей к общему числу друзей кандидата.

**Визуализация социального графа**  
Интерактивное отображение социальной сети: центральный пользователь (фиолетовый), текущие друзья (синие), рекомендованные кандидаты (зелёные). Толщина и цвет связей зависят от количества общих друзей.

**Фильтрация и статистика**  
- Исключение существующих друзей из рекомендаций
- Фильтр по минимальному количеству общих друзей
- Глобальная статистика по всей сети (охват, среднее количество рекомендаций)
- Детальный просмотр общих друзей для каждого кандидата

### Использование приложения

При запуске открывается окно с панелью управления, таблицей рекомендаций и визуализацией графа:

**Выбор пользователя**
1. В верхней панели выберите пользователя из выпадающего списка
2. Отобразится карточка пользователя с количеством друзей
3. Система автоматически выполнит анализ и построит рекомендации

**Просмотр рекомендаций**
1. В левой панели отобразится ТОП-10 кандидатов
2. Таблица показывает: номер, имя кандидата, количество общих друзей, коэффициент связности
3. Клик по строке таблицы показывает список общих друзей в правой панели

**Работа с графом**
1. В центральной области отображается визуализация социального графа
2. Центральный узел — выбранный пользователь (фиолетовый)
3. Синие узлы — текущие друзья
4. Зелёные узлы — рекомендованные кандидаты
5. Наведение на узел показывает подробную информацию


**Глобальная статистика**
1. Нажмите кнопку «Статистика сети» в верхней панели
2. Отобразится: общее количество пользователей, процент охвата, среднее количество рекомендаций

### Алгоритмы

**Поиск в ширину (BFS)**
```
1. Поместить целевого пользователя в очередь с расстоянием 0
2. Пока очередь не пуста:
   - Извлечь текущего пользователя и расстояние
   - Если расстояние = 2 → добавить в кандидаты
   - Если расстояние < 2 → добавить друзей в очередь
3. Отфильтровать существующих друзей
4. Отсортировать по количеству общих друзей
```

**Коэффициент связности**
```
coeff = mutualFriendsCount / candidateFriendsCount
```
где:
- `mutualFriendsCount` — количество общих друзей с целевым пользователем
- `candidateFriendsCount` — общее количество друзей у кандидата
